FlexConfig

FlexConfig: { globalConfig?: string; superRoot: string; trader: { deploy: { eversAll: number; eversAuth: number; minRefill: number; refillWallet: number }; order: { evers: number; mode: MakeOrderMode } }; web3?: ClientConfig }
  • Defined in flex/config.ts:27

Type declaration

  • Optional globalConfig?: string
  • superRoot: string
  • trader: { deploy: { eversAll: number; eversAuth: number; minRefill: number; refillWallet: number }; order: { evers: number; mode: MakeOrderMode } }
    • deploy: { eversAll: number; eversAuth: number; minRefill: number; refillWallet: number }
      • eversAll: number

        Full payment for Trader creation.

        remarks

        Must be specified in nanotokens, i.e. 1e9. Default value is 40e9.

      • eversAuth: number

        Payment for Auth Contract deploy. Included into eversAll.

        remarks

        Must be specified in nanotokens, i.e. 1e9. Default value is 1e9.

      • minRefill: number

        Minimal amount of EVERs the wallet receives from userIdIndex contract when a trade happens (when the wallet receives tokens) if the wallet's balance > refillWallet. Included into eversAll

        remarks

        Must be specified in nanotokens, i.e. 1e9. Default value is 0.1e9.

      • refillWallet: number

        When trader receives tokens the sum (refillWallet-wallet.eversBalance) is additionally sent to this wallet from userIdIndex contract. Included into eversAll.

        remarks

        Must be specified in nanotokens, i.e. 1e9. Default value is 10e9.

    • order: { evers: number; mode: MakeOrderMode }
      • evers: number
      • mode: MakeOrderMode
  • Optional web3?: ClientConfig

TradeInfo

TradeInfo: { amount: number; cursor: string; fees: number; feesToken: TokenInfo; liquidity: TradeLiquidity; pair: { address: string }; price: number; side: TradeSide; time: number }
  • Defined in flex/trader/types.ts:42

Type declaration

  • amount: number

    Amount of the major tokens

  • cursor: string
  • fees: number

    User fees for this trade. Measured in major tokens. If the user is a maker then fees is a value received by user as a bonus for making order. Note that in this case the fees is a negative value.

    If the user is a taker then fees is a value that the user pays to the exchange and maker.

  • feesToken: TokenInfo

    User fees token.

  • liquidity: TradeLiquidity

    Determines the users position in trade. Maker or taker. Maker is a trade counterparty whose order was earlier. Taker is a counterparty with a later order.

  • pair: { address: string }

    Flex market (pair)

    • address: string
  • price: number

    Price of the major token

  • side: TradeSide

    Determines the type of the later order (taker) in trade.

  • time: number

    Trade time as a unix time stamp
